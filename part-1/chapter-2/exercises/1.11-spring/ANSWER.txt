## Dockerfile

```dockerfile
FROM amazoncorretto:8

EXPOSE 8080

WORKDIR /usr/src/app

COPY . .

RUN ./mvnw package

CMD ["java", "-jar", "./target/docker-example-1.1.3.jar"]
```

## Command to Build and Run

Build the image:
```bash
cd spring-example-project
docker build -t spring-app .
```

Run the container:
```bash
docker run -d -p 8080:8080 spring-app
```

Or build and run in one command:
```bash
docker build -t spring-app . && docker run -d -p 8080:8080 spring-app
```

## Accessing the Application

Once the container is running, access the application at:
```
http://localhost:8080
```

You should see a page with a button labeled "Press here". Clicking it will display the success message.

## Explanation

The Dockerfile:

1. **FROM amazoncorretto:8** - Uses Amazon Corretto (Java 8) as the base image
2. **EXPOSE 8080** - Exposes port 8080 where the Spring application listens
3. **WORKDIR /usr/src/app** - Sets the working directory in the container
4. **COPY . .** - Copies the entire project from the host to the container
5. **RUN ./mvnw package** - Builds the application using Maven (mvnw is the Maven wrapper)
6. **CMD ["java", "-jar", "./target/docker-example-1.1.3.jar"]** - Runs the compiled JAR file

## Key Points

- **amazoncorretto:8** - Provides Java 8 runtime environment, as specified in the README
- **mvnw package** - Maven build command that creates the JAR file in the target directory
- **docker-example-1.1.3.jar** - The compiled application JAR file that gets executed
- Port 8080 is where the Spring Boot application runs by default

## Success Criteria

The exercise is completed when:
- The image builds successfully without errors
- The container starts and listens on port 8080
- The web page loads at http://localhost:8080 with the button interface
- Clicking the button displays the "Success" message
